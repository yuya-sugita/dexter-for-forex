# Polymarket 定量分析スキル

Renaissance（旧称 Sapiens）の Polymarket 移植版における主要ワーク
フロー。元ブランチの `trade-analysis` スキルを予測市場ドメインに
再構成したもので、**11思考プロトコル + 6エージェント**を Polymarket
文脈で発動させる。

## 基本原則（思考プロトコルからの継承）

このワークフローを実行する際は、以下のプロトコルを内部的に走らせ続ける:

- **Prompt 1（データが先）**: ナラティブ（「トランプは絶対勝つ」等）
  を凍結し、市場価格・ポール・ベースレートの数値からスタートする
- **Prompt 3（異邦人を雇う）**: 6人のエージェントを並行呼び出しし、
  単一視点に依存しない
- **Prompt 5（なぜを探すな）**: 市場が動いた理由を後付けで生成しない。
  パターンと条件付き確率のみで判断する
- **Prompt 6（シグナルの層）**: 最低3層の独立シグナルを要求
- **Prompt 9（モノリシック・モデル）**: 関連市場・ポール・ベースレートを
  常に文脈として併用
- **Prompt 11（美しさ）**: 特殊ケースのパッチ（「この市場だけ別ルール」）
  が必要になったら原理を再設計

## ツール呼び出しの基本原則

- **1ツールずつ順番に実行**し、結果を待ってから次へ進む。並列禁止
- 各ステップの結果をユーザーに都度報告する
- 数値・確率・条件付き分布で語り、ナラティブを生産しない

## ワークフローチェックリスト

```
Polymarket 定量分析:
- [ ] ステップ1: 市場の特定とスナップショット
- [ ] ステップ2: 暗黙確率と価格構造の抽出
- [ ] ステップ3: 事前分布（ベースレート + ポール）の構築
- [ ] ステップ4: 関連市場の整合性チェック
- [ ] ステップ5: 6エージェントの並行診断
- [ ] ステップ6: コンフルエンスの確認
- [ ] ステップ7: 期待値・ケリー比・破産確率の計算
- [ ] ステップ8: ポジション記録と継続観察計画
```

## ステップ1: 市場の特定とスナップショット

対象市場を特定し、現在の状態を把握する。

**ツール:** `get_market` (`market_id` または `slug`)

**抽出:**

- 現在の YES/NO 価格（バイナリ）または各候補の価格（マルチ）
- 24h 出来高、累積出来高、流動性
- 解決日、レゾルバ
- カテゴリ（politics / sports / crypto / economy / culture）

**判定:**

- 流動性が極端に低い（< $1k 出来高）→ 観察対象から外す（執行不可）
- 解決日が極端に近い（< 24h）→ ニュース駆動の最終局面、Prompt 8 の
  縮小モードを念頭に
- 解決日が極端に遠い（> 1年）→ ベースレート支配、ポール影響小

## ステップ2: 暗黙確率と価格構造の抽出

市場価格を確率に変換し、時間構造を見る。

**ツール:** `get_implied_probability`, `get_market_history`,
`get_term_structure`

**抽出:**

- 暗黙確率（手数料・スプレッド調整後）
- 過去30日の価格時系列とボラティリティ
- 解決日までの距離 vs 価格の構造
- 直近の価格急変（z-score > 2 or < -2）の有無

**判定:**

- 急変があった → ステップ5 で物理学者・暗号解読者を重点的に呼ぶ
- 価格が安定（低ボラ）→ 情報の織り込みが進んでいる、エッジは
  情報源統合（ステップ3）に偏る

## ステップ3: 事前分布（ベースレート + ポール）の構築

天文学者の流儀で事前分布を構築する。

**ツール:** `get_base_rate`, `get_polls`（選挙系のみ）

**抽出:**

- 類似イベントの過去ベースレート（例: 「与党大統領の再選率」「ホーム
  チームの地区優勝率」）
- ポール集計の現時点平均と分散（選挙系）
- 事前分布の平均と標準偏差

**判定:**

- 市場価格 vs ベースレートの差（30%以上の差は統計的に注目に値する）
- 市場価格 vs ポール平均の差（5%以上で精査対象）
- 事前分布が広い（不確実）→ 収縮推定で過反応を補正
- 事前分布が狭い（確実）→ 市場が事前分布から乖離していれば強いシグナル

## ステップ4: 関連市場の整合性チェック

数学者の流儀で同じイベントのサブ市場の整合性を見る。

**ツール:** `get_correlation_markets`, `calculate_arbitrage`

**抽出:**

- 関連サブ市場の価格一覧（候補別、トーナメント別）
- マルチアウトカム市場の合計価格（理論値 = 1.00、実測値 ≈ 1.00 ± 流動性誤差）
- 関連市場間のリターン相関（過去30日）

**判定:**

- 合計が 1.05 以上 → アービ機会（ベット側に売り手なし、買い手なし）
- 合計が 0.95 以下 → 流動性プレミアム or 全候補に賭ける機会
- 相関構造の異常（本来連動すべき市場が乖離）→ 情報漏洩の可能性

## ステップ5: 6エージェントの並行診断

これがこのスキルの**核心**。各エージェントを独立に呼び出し、それぞれの
原語で診断を取得する。

**呼び出すスキル（並行ではなく順次、しかし独立に）:**

1. **`outsider-mathematician`** — 関連市場の合計、共和分残差、固有値
   分解で構造を確認
2. **`outsider-physicist`** — 価格急変が相転移か通常変動か、ボラ
   レジームを判定
3. **`outsider-astronomer`** — ベースレートとポールから事前分布を
   構築し、市場価格との乖離を収縮推定
4. **`outsider-speech-recognition`** — ニュースフローの隠れ状態系列を
   復号、センチメントの遷移を推定
5. **`outsider-cryptanalyst`** — 大口保有者の動き、急な板の動き、
   ニュース前の価格変化から情報漏洩を検出
6. **`quant-analyst`** — 上記5人の診断を入力として受け取り、ケリー比・
   期待値・破産確率に翻訳

**重要な制約（Prompt 3 × Prompt 9）:**

- 各エージェントは金融語彙を使わず（クオンツアナリスト除く）、それぞれの
  原語で診断する
- クオンツアナリストは**単独で動かない**——5人の異邦人の診断が揃って
  から呼ぶ
- 5人の診断が散らばった場合（コンフルエンスが取れない）、クオンツ
  アナリストも判断を保留する

## ステップ6: コンフルエンスの確認

6人の診断を突合し、判断可能か判定する。

**チェック:**

- 5人の異邦人のうち、何人が**同じ方向**（買い/売り/中立）を示したか
- 4人以上一致 → 強いコンフルエンス、執行候補
- 2-3人一致 → 弱いシグナル、サイズを α=0.5 に縮小
- 1人以下、または全員意見が分かれた → **判断保留**（Prompt 3 違反を
  避ける）

**Prompt 6 への接続:**

- 各エージェントの診断が独立シグナルとして機能している（同じ情報の
  焼き直しでない）かを確認
- 独立性が低い場合は層数を実質的に減らして評価する

## ステップ7: 期待値・ケリー比・破産確率の計算

クオンツアナリストの執行翻訳を実行する。

**ツール:** `calculate_expected_value`, `calculate_kelly_binary`,
`calculate_position_size`, `calculate_risk_of_ruin`,
`check_bankroll_health`

**計算:**

- **推定真確率 p**: 6エージェント診断から導出（収縮推定後）
- **市場価格 q**: ステップ2 から
- **エッジ**: |p - q|（5%以上が望ましい）
- **ケリー比**: f* = (p - q) / (1 - q)（買い）または (q - p) / q（売り）
- **使用するケリー比**: f* / 4（quarter-Kelly、Polymarket 流動性と
  解決リスクを考慮）
- **ポジションサイズ**: bankroll × ケリー比
- **破産確率**: 100ベットでバンクロール 50% を割る確率
- **集中度**: 既存ポジションとの相関考慮

**判定:**

- エッジ < 5% → スキップ（執行コスト・解決リスクで消える）
- 破産確率 > 1% → サイズを縮小（Prompt 8）
- 集中度が既に高い → 別カテゴリの市場を優先

## ステップ8: ポジション記録と継続観察計画

執行決定を記録し、継続観察計画を立てる。

**ツール:** `record_position`

**記録項目:**

```
market_id: <UUID または slug>
outcome: YES / NO / <候補名>
entry_price: <価格>
size: <USDC>
rationale: <Prompt 5 準拠の構造化記録>
  - 6エージェント診断の要約（事後ナラティブではなく数値）
  - 推定真確率 vs 市場価格
  - 使用したケリー比
  - 想定解決シナリオの確率分布
notes: [POSTMORTEM] テンプレートの準備（Prompt 7）
  - facts: 入力時刻・価格・サイズ
  - hypothesis: 「真確率は X%、市場は Y%、Z までに収束する」
  - failure_mode: pending（解決時に記入）
  - new_rule: pending
```

**継続観察計画:**

- 解決日まで何回チェックするか
- ニュースイベントのカレンダー登録
- ポジション縮小の発動条件（Prompt 8）
- 解決後のポストモーテム実施日

## ワークフロー終了後の必須事項

1. ユーザーに6エージェントのコンフルエンス状況を報告
2. 最終的な期待値とケリー比を明示
3. 採用されたポジションのリスク総量を提示
4. **「なぜ」の問いには「データがこう言っている」で答える**
   （Prompt 5）

## 関連スキル

- 個別エージェントの診断: `quant-analyst`, `outsider-mathematician`,
  `outsider-physicist`, `outsider-astronomer`,
  `outsider-speech-recognition`, `outsider-cryptanalyst`
- バンクロール管理: `risk-management`

## このスキルが守る思考プロトコル

このスキル全体は、以下のプロトコルを Polymarket 文脈で発動させる:

| プロトコル | このスキルでの発動箇所 |
|---|---|
| Prompt 1: データが先 | ステップ1-2（市場データから開始） |
| Prompt 2: 51%エッジ | ステップ7（small edge × high frequency） |
| Prompt 3: 異邦人を雇う | ステップ5（6エージェント並行） |
| Prompt 5: なぜを探すな | ステップ8（rationale を数値で記録） |
| Prompt 6: シグナルの層 | ステップ5-6（独立層のコンフルエンス） |
| Prompt 7: 失敗をデータに | ステップ8（POSTMORTEM テンプレート） |
| Prompt 8: 縮小プロトコル | ステップ7（破産確率 > 1% で縮小） |
| Prompt 9: モノリシック | ステップ3-4（関連市場・ベースレート統合） |
| Prompt 10: 尖度 | ステップ5-6（6層の結合分布で生きる） |
| Prompt 11: 美しさ | スキル全体（特殊ケース禁止、原理的判断） |

各ステップで該当プロトコルが内部的に走っていることを意識する。
